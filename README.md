# bookstore-sql-project
A PostgreSQL-based Bookstore database project demonstrating data retrieval, joins, and aggregate queries for books, customers, and orders.

#  Bookstore SQL Project

## Description
This project demonstrates SQL query skills using a **Bookstore database** built in **PostgreSQL**.  
It focuses on data retrieval, filtering, aggregation, and analysis based on books, customers, and orders.

## Tables Used
- Books
- Customers
- Orders

## Queries Covered
The project includes queries to:
- Retrieve books by genre and publication year
- List customers by country
- Filter orders by date and amount
- Calculate total revenue and remaining stock
- Find most expensive and most ordered books
- Analyze customer orders and spending
- Perform genre-wise and author-wise sales analysis

## Joins Used
- **2-table JOINs**
  - Books ↔ Orders
  - Customers ↔ Orders
- **JOIN types:** INNER JOIN, LEFT JOIN

## SQL Concepts Used
- SELECT, WHERE
- JOIN (INNER, LEFT)
- GROUP BY, HAVING
- Aggregate functions (SUM, COUNT, AVG)
- Date filtering
- Constraints handling

## Database
PostgreSQL | SQL | pgAdmin
